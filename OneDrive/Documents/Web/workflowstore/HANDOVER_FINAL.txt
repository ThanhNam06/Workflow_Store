4) Important endpoints (summary)
Auth:
- POST /api/auth/register
- POST /api/auth/login (rate-limited)

Admin (x-admin-key header required or Authorization: Bearer <ADMIN_API_KEY>):
- POST /api/admin/workflows  (multipart/form-data, file field: 'file')  => create workflow + save uploaded file to backend/data/workflows
- PUT /api/admin/workflows/:id
- DELETE /api/admin/workflows/:id  => deletes workflow and removes file from disk
- GET /api/admin/orders
- POST /api/admin/orders/:id/mark-paid  => mark paid + send email + create tokens

Orders & Payments:
- POST /api/orders  => create order
- POST /api/create-checkout-session  => create Stripe session (USD)
- POST /api/webhooks/stripe  => Stripe webhook (verify using STRIPE_WEBHOOK_SECRET)
- POST /api/webhooks/vietqr  => VietQR webhook; now expects Authorization: Apikey <VIETQR_WEBHOOK_SECRET>

Downloads:
- GET /api/orders/:id/downloads  => return token-based URLs (requires admin OR order accessToken OR owner JWT)
- GET /api/download/:token  => stream file if token valid (rate-limited). Consumes token remaining.

5) Security & operational features
- Download tokens: TTL 24h, remaining 3 downloads, persisted to backend/data/db.json
- Maintenance job: startMaintenanceLoop(db) runs cleanup at startup and every 1 hour to remove expired/exhausted tokens
- Rate limiting (express-rate-limit):
  - /api/download/:token  => 100 req/IP/hour
  - /api/auth/login        => 20 req/IP/15min
  - /api/admin/*          => 200 req/IP/hour
- Blocklist: in-memory blocklist; exhausted token accesses add IP to blocklist. For production, integrate persistent blocklist or Fail2ban.
- PM2 usage: run backend with pm2 (pm2 start src/index.js --name workflowstore-backend --update-env) and pm2 save

6) Quick deploy checklist
- Clone repo, checkout tag v1.0.0
- Create backend/.env with keys above
- Build frontend: cd frontend && npm ci && npm run build
- Install backend deps: cd backend && npm ci
- Start backend: pm2 start src/index.js --name workflowstore-backend --update-env
- Configure Nginx reverse proxy and SSL
- Configure Stripe webhooks and VietQR provider (Authorization: Apikey <secret>)

7) Notes
- The app expects workflows files under backend/data/workflows. Clean up old files manually or via scripts.
- Back up db.json regularly.
